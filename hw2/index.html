Overview

In this assignment I implemented a mesh editor built around the half-edge data structure. Starting from 1-D Bézier curves and extending to 3-D Bézier surfaces, I then moved into the half-edge mesh itself to implement smooth vertex normals, edge flip, edge split, and finally Loop subdivision for progressive mesh refinement.

The most interesting thing I took away from this project is how elegantly the half-edge structure encodes local mesh topology — every operation (flip, split, subdivision) reduces to carefully re-wiring a small, fixed set of pointers. Getting those reassignments exactly right required drawing detailed diagrams, but once correct the operations compose cleanly to produce the subdivision algorithm.

Part 1 — Bézier Curves (de Casteljau's Algorithm)

De Casteljau's algorithm evaluates a Bézier curve of degree n by repeatedly linearly interpolating between consecutive control points at parameter t. Each application reduces the number of points by one, until a single point remains — the point on the curve at parameter t.

My implementation iterates over each consecutive pair of points in the current level and computes lerp(p[i], p[i+1], t), storing the results as the next level. This single step is called repeatedly until one point remains.

Part 2 — Bézier Surfaces (de Casteljau in 3-D)

The algorithm extends to surfaces by operating on an n × n grid of control points with two parameters u and v. For each row of control points, de Casteljau is run with parameter u to produce a single "moving curve" point. The n resulting points are then fed into one more round of de Casteljau with parameter v to yield the surface point at (u, v).

My implementation reuses the same 1-D helper from Part 1 twice: once per row to reduce each row to a point, and once more across those points to get the final surface position.

Part 3 — Area-Weighted Vertex Normals

To compute the area-weighted vertex normal, I iterate over all faces incident to the vertex using the half-edge data structure. Starting from the vertex's halfedge, I repeatedly follow h = h->twin()->next() to circulate around the vertex. For each non-boundary face, I compute the face normal using the cross product of two triangle edges. Because the magnitude of the cross product is proportional to the triangle's area, this automatically produces area weighting. I accumulate these face normals and finally normalize the sum to obtain a unit vertex normal.

Part 4 — Edge Flip

To implement the edge flip operation, I first collected all relevant halfedges, vertices, edges, and faces for the two triangles sharing the edge. I then reassigned all pointers using setNeighbors to rewire the mesh so the shared edge connects the two previously non-adjacent vertices instead. No new mesh elements are created or deleted — only pointers are reassigned.

The key insight was to be exhaustive: every halfedge, vertex, edge, and face has its pointers reset even if unchanged, which prevents subtle bugs from stale pointer states.

Part 5 — Edge Split

To implement edge split, I collected all existing halfedges, vertices, edges, and faces around the shared edge, then created one new vertex at the midpoint, three new edges, two new faces, and six new halfedges. I then used setNeighbors to wire everything into four triangles.

The trickiest part was carefully deriving the correct tip vertex for each halfedge and ensuring all four face windings were consistent. I also made sure vertex halfedge() pointers point to halfedges that originate from that vertex.

Part 6 — Loop Subdivision

Loop subdivision is implemented in four steps:

  1. New positions for all old vertices are computed using the Loop weighting formula before any topology changes.
  2. New positions for edge midpoints are computed using the 3/8 and 1/8 weights from the four surrounding vertices.
  3. Every original edge is split and the new midpoint vertices are placed at the precomputed positions.
  4. Any new edge connecting an old and new vertex is flipped.

Sharp corners & edges

Sharp corners and edges get smoothed out with each subdivision step because the Loop weights pull vertices toward their neighbors. Pre-splitting edges near sharp features can help preserve them by giving the subdivision more geometry to work with in those areas.

Asymmetric cube subdivision

When applying loop subdivision to dae/cube.dae, the result becomes asymmetric after repeated subdivisions. This happens because each face of the cube has a single diagonal edge — the two triangles on each face are not mirror images of each other — and this asymmetry gets amplified with each step.

By pre-processing the cube and splitting the diagonal on each face so that each face has an X shape, the mesh gains the symmetry needed to subdivide evenly.
